## 说明与统一假设

为了让所有数值题和题图、强化学习课件保持一致，我采用如下统一假设：

1. 除非题目另外说明，取 `gamma = 1`。
2. 对于 Q2、Q3、Q5：
   - 环境为一个确定性 `3 x 4` 的 grid world，状态编号按行优先排列：
     - 第一行：`0 1 2 3`
     - 第二行：`4 5 6 7`
     - 第三行：`8 9 10 11`
   - 状态 `3` 是终止状态，终止值为 `0`
   - 状态 `7` 是终止状态，终止值为 `-1`
   - 状态 `4` 是障碍物
   - 每走一步的代价都是 `-0.1`
   - 如果动作越界，或者撞到障碍物，则机器人留在原状态
3. 对于 Q3，如果一个格子里画了多个箭头，则把它看成在这些箭头对应动作之间等概率随机选择的策略。
4. 对于 Q1：
   - 图中给出的 `8` 条序列都视为样本 episode
   - Monte Carlo 按课件中的 `first-visit MC` 计算
   - TD 按题图自然阅读顺序做在线更新：上排从左到右，再下排从左到右
   - 初始值取 `V(A)=0`、`V(B)=0`

## Q1

题目：

> 已知若干条 state-reward sequence，分别用 Monte Carlo 和 TD 方法估计状态 A 与状态 B 的 value function。设 `alpha = 1`，`gamma = 1`。

### 1. Monte Carlo 解法

8 个 episode 为：

1. `A-0-B-1-A-1-B-0`
2. `A-1-B-1-A-1-B-1`
3. `A-0-B-1-A-1`
4. `A-0-B-1-A-1-B-0`
5. `B-0-A-1-B-0`
6. `B-0-A-1-B-0`
7. `B-1-B-1`
8. `B-0`

按 first-visit Monte Carlo：

对于 `A`，只统计每个 episode 中第一次访问 `A` 后的回报：

- Ep1: `0+1+1+0 = 2`
- Ep2: `1+1+1+1 = 4`
- Ep3: `0+1+1 = 2`
- Ep4: `0+1+1+0 = 2`
- Ep5: `1+0 = 1`
- Ep6: `1+0 = 1`

所以：

```text
V_MC(A) = (2+4+2+2+1+1)/6 = 2
```

对于 `B`，统计每个 episode 中第一次访问 `B` 后的回报：

- Ep1: `1+1+0 = 2`
- Ep2: `1+1+1 = 3`
- Ep3: `1+1 = 2`
- Ep4: `1+1+0 = 2`
- Ep5: `0+1+0 = 1`
- Ep6: `0+1+0 = 1`
- Ep7: `1+1 = 2`
- Ep8: `0`

所以：

```text
V_MC(B) = (2+3+2+2+1+1+2+0)/8 = 13/8 = 1.625
```

### 2. TD(0) 解法

TD(0) 在线更新公式为：

```text
V(s) <- r + gamma * V(s')
```

因为这里 `alpha = 1`，`gamma = 1`。

初始值：

- `V(A)=0`
- `V(B)=0`

#### Ep1：`A-0-B-1-A-1-B-0`

- `A ->(0)-> B`
  - `V(A) = 0 + V(B) = 0`
- `B ->(1)-> A`
  - `V(B) = 1 + V(A) = 1`
- `A ->(1)-> B`
  - `V(A) = 1 + V(B) = 2`
- `B ->(0)-> terminal`
  - `V(B) = 0`

此时：

- `V(A)=2`
- `V(B)=0`

#### Ep2：`A-1-B-1-A-1-B-1`

- `A ->(1)-> B`
  - `V(A) = 1 + V(B) = 1`
- `B ->(1)-> A`
  - `V(B) = 1 + V(A) = 2`
- `A ->(1)-> B`
  - `V(A) = 1 + V(B) = 3`
- `B ->(1)-> terminal`
  - `V(B) = 1`

此时：

- `V(A)=3`
- `V(B)=1`

#### Ep3：`A-0-B-1-A-1`

- `A ->(0)-> B`
  - `V(A) = 0 + V(B) = 1`
- `B ->(1)-> A`
  - `V(B) = 1 + V(A) = 2`
- `A ->(1)-> terminal`
  - `V(A) = 1`

此时：

- `V(A)=1`
- `V(B)=2`

#### Ep4：`A-0-B-1-A-1-B-0`

- `A ->(0)-> B`
  - `V(A) = 0 + V(B) = 2`
- `B ->(1)-> A`
  - `V(B) = 1 + V(A) = 3`
- `A ->(1)-> B`
  - `V(A) = 1 + V(B) = 4`
- `B ->(0)-> terminal`
  - `V(B) = 0`

此时：

- `V(A)=4`
- `V(B)=0`

#### Ep5：`B-0-A-1-B-0`

- `B ->(0)-> A`
  - `V(B) = 0 + V(A) = 4`
- `A ->(1)-> B`
  - `V(A) = 1 + V(B) = 5`
- `B ->(0)-> terminal`
  - `V(B) = 0`

此时：

- `V(A)=5`
- `V(B)=0`

#### Ep6：`B-0-A-1-B-0`

- `B ->(0)-> A`
  - `V(B) = 0 + V(A) = 5`
- `A ->(1)-> B`
  - `V(A) = 1 + V(B) = 6`
- `B ->(0)-> terminal`
  - `V(B) = 0`

此时：

- `V(A)=6`
- `V(B)=0`

#### Ep7：`B-1-B-1`

- `B ->(1)-> B`
  - `V(B) = 1 + V(B) = 1`
- `B ->(1)-> terminal`
  - `V(B) = 1`

此时：

- `V(A)=6`
- `V(B)=1`

#### Ep8：`B-0`

- `B ->(0)-> terminal`
  - `V(B) = 0`

最终 TD 估计值：

- `V_TD(A) = 6`
- `V_TD(B) = 0`

### Q1 最终答案

- Monte Carlo：
  - `V(A) = 2`
  - `V(B) = 1.625`
- TD：
  - `V(A) = 6`
  - `V(B) = 0`

## Q2

题目：

> 给定如下环境配置，请计算所有状态的最终 value function。（以状态 0 和状态 2 为例写出计算过程；每一步代价为 `-0.1`）

### Bellman 最优方程

对每个非终止状态：

```text
V*(s) = -0.1 + max_a V*(s')
```

其中 `s'` 是执行动作 `a` 后到达的下一状态。

终止状态：

- `V*(3) = 0`
- `V*(7) = -1`

障碍物：

- 状态 `4` 不可进入

### 例 1：状态 2

从状态 `2` 出发，可能的下一状态：

- 向上：仍留在 `2`
- 向下：到达 `6`
- 向左：到达 `1`
- 向右：到达终止状态 `3`

因此：

```text
V*(2) = -0.1 + max{V*(2), V*(6), V*(1), V*(3)}
      = -0.1 + max{-0.1, -0.2, -0.2, 0}
      = -0.1
```

### 例 2：状态 0

从状态 `0` 出发，可能的下一状态：

- 向上：仍留在 `0`
- 向左：仍留在 `0`
- 向下：撞到障碍物 `4`，所以仍留在 `0`
- 向右：到达 `1`

因此：

```text
V*(0) = -0.1 + max{V*(0), V*(0), V*(0), V*(1)}
      = -0.1 + max{-0.3, -0.3, -0.3, -0.2}
      = -0.3
```

### 所有状态最终值

```text
---------------------------------
|  0:-0.3 |  1:-0.2 |  2:-0.1 | 3: 0   |
---------------------------------
|  4:wall |  5:-0.3 |  6:-0.2 | 7:-1   |
---------------------------------
|  8:-0.5 |  9:-0.4 | 10:-0.3 | 11:-0.4|
---------------------------------
```

所以 Q2 的最终答案为：

- `V*(0) = -0.3`
- `V*(1) = -0.2`
- `V*(2) = -0.1`
- `V*(3) = 0`
- `V*(4) = wall / 不适用`
- `V*(5) = -0.3`
- `V*(6) = -0.2`
- `V*(7) = -1`
- `V*(8) = -0.5`
- `V*(9) = -0.4`
- `V*(10) = -0.3`
- `V*(11) = -0.4`

## Q3

题目：

> 对于下列环境配置、初始 value function 和给定策略，请对所有状态做 1 轮 policy evaluation。（以状态 0 和状态 2 为例写出计算过程；每一步代价为 `-0.1`）

### 初始值

由题图给出的初始状态值为：

```text
---------------------------------
|  0:-0.1 |  1:-0.1 |  2:-0.1 | 3: 0   |
---------------------------------
|  4:wall |  5:-0.1 |  6:-0.1 | 7:-1   |
---------------------------------
|  8:-0.1 |  9:-0.1 | 10:-0.1 | 11:-0.1|
---------------------------------
```

### 题图对应策略

我把图里的箭头解释成在所画动作之间等概率随机选择的策略：

- `pi(0) = {L, U, R}`，每个动作概率 `1/3`
- `pi(1) = {L, U, R, D}`，每个动作概率 `1/4`
- `pi(2) = {R}`，概率 `1`
- `pi(5) = {U, R, D}`，每个动作概率 `1/3`
- `pi(6) = {L, U, D}`，每个动作概率 `1/3`
- `pi(8) = {L, R, D}`，每个动作概率 `1/3`
- `pi(9) = {L, U, R, D}`，每个动作概率 `1/4`
- `pi(10) = {L, U, R, D}`，每个动作概率 `1/4`
- `pi(11) = {L, R, D}`，每个动作概率 `1/3`

### 1 轮 Policy Evaluation

使用公式：

```text
V_1(s) = -0.1 + sum_a pi(a|s) * V_0(s')
```

其中 `s'` 是动作 `a` 执行后的下一状态。

### 例 1：状态 0

从状态 `0` 出发：

- `L` -> 留在 `0`
- `U` -> 留在 `0`
- `R` -> 到达 `1`

所以：

```text
V_1(0) = -0.1 + (1/3)[V_0(0) + V_0(0) + V_0(1)]
       = -0.1 + (1/3)[-0.1 + -0.1 + -0.1]
       = -0.2
```

### 例 2：状态 2

从状态 `2` 出发，策略只选择 `R`，而 `R` 会到达终止状态 `3`。

所以：

```text
V_1(2) = -0.1 + V_0(3)
       = -0.1
```

### 1 轮更新后的状态值

```text
---------------------------------
|  0:-0.2 |  1:-0.2 |  2:-0.1 | 3: 0   |
---------------------------------
|  4:wall |  5:-0.2 |  6:-0.2 | 7:-1   |
---------------------------------
|  8:-0.2 |  9:-0.2 | 10:-0.2 | 11:-0.2|
---------------------------------
```

因此 Q3 的结果为：

- `V_1(0) = -0.2`
- `V_1(1) = -0.2`
- `V_1(2) = -0.1`
- `V_1(3) = 0`
- `V_1(4) = wall / 不适用`
- `V_1(5) = -0.2`
- `V_1(6) = -0.2`
- `V_1(7) = -1`
- `V_1(8) = -0.2`
- `V_1(9) = -0.2`
- `V_1(10) = -0.2`
- `V_1(11) = -0.2`

## Q4

题目：

> 分别画出 SARSA 和 Q-learning 的系统框图，并比较两种方法在智能机器人运动规划中的差异（行为策略与估计策略、探索与利用能力、收敛速度、最优值估计能力、最优策略估计能力等）。

### 1. SARSA 系统图

```text
当前状态 s
  |
  v
按行为策略 pi 选择动作 a
  |
  v
观察到奖励 r 和下一状态 s'
  |
  v
按同一策略 pi 选择下一动作 a'
  |
  v
Q(s,a) <- Q(s,a) + alpha [r + gamma Q(s',a') - Q(s,a)]
```

### 2. Q-learning 系统图

```text
当前状态 s
  |
  v
按行为策略选择动作 a
  |
  v
观察到奖励 r 和下一状态 s'
  |
  v
Q(s,a) <- Q(s,a) + alpha [r + gamma max_a' Q(s',a') - Q(s,a)]
```

### 3. 严格按题目要求的对比表

| 对比维度 | SARSA | Q-learning |
|---|---|---|
| 行为策略 (behavior policy) | `on-policy`，行为策略就是学习策略。 | 行为策略可带探索。 |
| 估计策略 (estimation policy) | 用实际选到的下一动作 `a'` 更新。 | 用 `max_a Q(s',a)` 更新。 |
| 探索能力 (capability of exploration) | 可以探索，且更新会反映探索结果。 | 可以探索，但更新目标始终偏向贪心。 |
| 利用能力 (capability of exploitation) | 更保守。 | 更激进。 |
| 收敛速度 (convergence speed) | 一般较稳，但可能略慢。 | 常较快，但可能波动更大。 |
| 最优价值估计能力 (capability for optimal value estimation) | 主要估计当前策略的 `Q^\pi`。 | 可直接逼近最优值 `Q^*`。 |
| 最优策略估计能力 (capability for optimal policy estimation) | 需逐渐贪心化后才逼近最优策略。 | 更直接导出最优贪心策略。 |
| 对机器人运动规划的意义 | 更安全、更保守。 | 更直接追求最优路径。 |

### 4. 在机器人运动规划中的总结

- `SARSA`：更安全、更保守。
- `Q-learning`：更直接、更偏最优。

## Q5

题目：

> 对于 [3] 中的问题，Q-learning 的最终 Q-table 是什么？（输入为状态-动作对，输出为 Q 值；以状态 0 和状态 2 为例）

### Q-learning 目标

在确定性环境下：

```text
Q*(s,a) = -0.1 + V*(s')
```

其中 `s'` 是动作 `a` 执行后的下一状态，`V*` 来自 Q2 的最优状态值。

### 例 1：状态 0

由 Q2 可知：

- `V*(0) = -0.3`
- `V*(1) = -0.2`

状态 `0` 可执行的动作：

- `U`：留在 `0`
- `D`：撞到障碍物 `4`，仍留在 `0`
- `L`：留在 `0`
- `R`：到达 `1`

所以：

```text
Q*(0,U) = -0.1 + V*(0) = -0.4
Q*(0,D) = -0.1 + V*(0) = -0.4
Q*(0,L) = -0.1 + V*(0) = -0.4
Q*(0,R) = -0.1 + V*(1) = -0.3
```

### 例 2：状态 2

由 Q2 可知：

- `V*(2) = -0.1`
- `V*(1) = -0.2`
- `V*(6) = -0.2`
- `V*(3) = 0`

状态 `2` 可执行的动作：

- `U`：留在 `2`
- `D`：到达 `6`
- `L`：到达 `1`
- `R`：到达终止状态 `3`

所以：

```text
Q*(2,U) = -0.1 + V*(2) = -0.2
Q*(2,D) = -0.1 + V*(6) = -0.3
Q*(2,L) = -0.1 + V*(1) = -0.3
Q*(2,R) = -0.1 + V*(3) = -0.1
```

### 最终 Q-table

| 状态 | `Q(U)` | `Q(D)` | `Q(L)` | `Q(R)` |
|---|---:|---:|---:|---:|
| 0 | -0.4 | -0.4 | -0.4 | -0.3 |
| 1 | -0.3 | -0.4 | -0.4 | -0.2 |
| 2 | -0.2 | -0.3 | -0.3 | -0.1 |
| 5 | -0.3 | -0.5 | -0.4 | -0.3 |
| 6 | -0.2 | -0.4 | -0.4 | -1.1 |
| 8 | -0.6 | -0.6 | -0.6 | -0.5 |
| 9 | -0.4 | -0.5 | -0.6 | -0.4 |
| 10 | -0.3 | -0.4 | -0.5 | -0.5 |
| 11 | -1.1 | -0.5 | -0.4 | -0.5 |

特殊状态说明：

- 状态 `3`：终止状态
- 状态 `4`：障碍物
- 状态 `7`：终止状态

### 从 Q-table 看出的贪心最优动作

- `0 -> R`
- `1 -> R`
- `2 -> R`
- `5 -> U` 或 `R`
- `6 -> U`
- `8 -> R`
- `9 -> U` 或 `R`
- `10 -> U`
- `11 -> L`
